# namespace и cgroups
__namespace__ (пространство имен) - это механизм ядра Linux, который позволяет создавать изолированные экземпляры глобальных системных ресурсов для групп процессов. Процессы внутри одного namespace «видят» свой набор этих ресурсов, изолированный от других namespaces, хотя физически ресурсы могут быть общими.
### Типы namespace
|Тип    |Изолируемый ресурс
|:------|:-----------------|
|PID    |Идентификаторы процессов (процесс может иметь PID 1 внутри namespace, а на хосте — 2345)|
|NET    |Сетевой стек — интерфейсы, маршруты, правила iptables, порты|
|MNT    |Точки монтирования файловых систем (видит свой /, не видит хостовые монтирования)|
|UTS    |Имя хоста и доменное имя (каждый namespace может иметь своё hostname)|
|IPC    |Межпроцессное взаимодействие (очереди сообщений, разделяемая память, семафоры System V)|
|USER   |Пользователи и группы — позволяет процессу иметь root внутри namespace, не имея привилегий на хосте|
|CGROUP |	Контрольные группы — виртуализирует пути к cgroup, изолирует правила ограничения ресурсов|
|TIME   |Системное время — возможность иметь своё смещение времени|
### Как namespaces применяются в Docker
При запуске контейнера Docker (через runc) создаётся набор __namespaces__:
1.  PID — процесс приложения внутри получает PID 1, не видит процессов хоста.
2.  NET — отдельный сетевой стек с виртуальным интерфейсом eth0.
3.  MNT — своё корневое устройство (chroot/pivot_root на образ контейнера).
4.  UTS — можно задать своё hostname контейнера.
5.  IPC — изоляция очередей сообщений между контейнерами.
6.  USER (опционально) — при запуске в --userns=remap режиме «rootless».
7.  CGROUP — чтобы контейнер «видел» только свои квоты на CPU/RAM.
После создания namespaces Docker запускает процесс внутри них — сам контейнер выполняется в userspace, но с перенаправленными пространствами имён.
### cgroups
__cgroups__ (сокращение от control groups) - это механизм ядра Linux, который ограничивает, учитывает и изолирует использование системных ресурсов (CPU, память, дисковый ввод-вывод, сеть) для набора процессов. В отличие от __namespaces__, которые обеспечивают _изоляцию_ (процесс видит изолированное окружение), __cgroups__ обеспечивают _контроль_ (сколько ресурсов можно потребить). 
### Устройство и основные концепции
1.  __Иерархия__: Cgroups организованы в древовидную структуру (аналогично файловой системе). Ребёнок наследует ограничения родителя, но может ужесточать их, но не ослаблять. Например, если родительской группе выделено 2 ядра CPU, дочерняя не может получить 4.
2.  __Контроллеры (подсистемы)__: <br>Каждый контроллер отвечает за определённый тип ресурса:<br> 
    |Контроллер |Функция    |
    |:----------|:----------|
    |cpu|Ограничение доступа к процессору (доля времени, жесткая граница в процентах)|
    |memory|Лимит на оперативную память и swap; при превышении — OOM-killer|
    |blkio   |Лимит на операции ввода-вывода с диском (чтение/запись в байтах или операциях в секунду)|
    |pids|Максимальное количество процессов/потоков|
    |net_cls / net_prio|Маркировка сетевых пакетов и приоритет трафика|
    |cpuset|Привязка процесса к конкретным ядрам CPU или узлам NUMA|
    |freezer|Приостановка или возобновление всех процессов в группе|
    |devices|Разрешение/запрет доступа к файлам устройств|
3.  __Интерфейс через псевдо-файловую систему__:<br>Управление __cgroups__ производится через виртуальную файловую систему __cgroupfs__, обычно смонтированную в _/sys/fs/cgroup/_. Создание новой группы — это создание подкаталога __mkdir__. Установка лимита — запись значения в файл внутри этого каталога.
## Для чего же нужны пространства и группы
В работе Docker-контейнера __namespaces__ и __cgroups__ являются основными механизмами жизнедеятельности узлов. 
1. __Namespace__ - для логической изоляции.
2. __Cgroups__ - ограничение ресурсов, недопускающее их исчерпывания.
# Docker
__Docker__ - это программная платформа для автоматизации развертывания приложений внутри изолированных сред, называемых контейнерами. Она использует уровень виртуализации на уровне операционной системы (operating system–level virtualization), где ядро хоста разделяется между несколькими пользовательскими пространствами.
# Контейнер
__Контейнер__ - это легковесная, переносимая единица исполнения приложения, которая объединяет исполняемый код, среду выполнения, системные библиотеки и настройки. В отличие от виртуальной машины, контейнер не эмулирует аппаратное обеспечение и не запускает полноценную гостевую ОС. Вместо этого он использует механизмы ядра Linux (такие как пространства имен — namespaces для изоляции процессов, сети, файловой системы, и контрольные группы — cgroups для ограничения ресурсов), что обеспечивает минимальные накладные расходы и быстрое время запуска.
