# namespace и cgroups
__namespace__ (пространство имен) - это механизм ядра Linux, который позволяет создавать изолированные экземпляры глобальных системных ресурсов для групп процессов. Процессы внутри одного namespace «видят» свой набор этих ресурсов, изолированный от других namespaces, хотя физически ресурсы могут быть общими.
## Типы namespace
|Тип    |Изолируемый ресурс
|:------|:-----------------|
|PID    |Идентификаторы процессов (процесс может иметь PID 1 внутри namespace, а на хосте — 2345)|
|NET    |Сетевой стек — интерфейсы, маршруты, правила iptables, порты|
|MNT    |Точки монтирования файловых систем (видит свой /, не видит хостовые монтирования)|
|UTS    |Имя хоста и доменное имя (каждый namespace может иметь своё hostname)|
|IPC    |Межпроцессное взаимодействие (очереди сообщений, разделяемая память, семафоры System V)|
|USER   |Пользователи и группы — позволяет процессу иметь root внутри namespace, не имея привилегий на хосте|
|CGROUP |	Контрольные группы — виртуализирует пути к cgroup, изолирует правила ограничения ресурсов|
|TIME   |Системное время — возможность иметь своё смещение времени|
# Docker
__Docker__ - это программная платформа для автоматизации развертывания приложений внутри изолированных сред, называемых контейнерами. Она использует уровень виртуализации на уровне операционной системы (operating system–level virtualization), где ядро хоста разделяется между несколькими пользовательскими пространствами.
# Контейнер
__Контейнер__ - это легковесная, переносимая единица исполнения приложения, которая объединяет исполняемый код, среду выполнения, системные библиотеки и настройки. В отличие от виртуальной машины, контейнер не эмулирует аппаратное обеспечение и не запускает полноценную гостевую ОС. Вместо этого он использует механизмы ядра Linux (такие как пространства имен — namespaces для изоляции процессов, сети, файловой системы, и контрольные группы — cgroups для ограничения ресурсов), что обеспечивает минимальные накладные расходы и быстрое время запуска.
